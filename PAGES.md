# 페이지 기능 정리

## 라우트 구조

| 경로 | 페이지 | 접근 권한 |
|------|--------|-----------|
| `/` | 상품 목록 | 전체 공개 |
| `/product/:id` | 상품 상세 | 전체 공개 |
| `/login` | 로그인 | 비로그인 전용 |
| `/signup` | 회원가입 | 비로그인 전용 |
| `/cart` | 장바구니 | 로그인 필요 |
| `/payment` | 결제 | 로그인 필요 |
| `/payment/success` | 주문 완료 | 로그인 필요 |
| `/orders` | 내 주문 내역 | 로그인 필요 |
| `/admin/inventory` | 재고 관리 | 관리자 전용 |
| `/admin/orders` | 주문 관리 | 관리자 전용 |

---

## 1. 상품 목록 페이지 `/`

**파일:** `src/pages/ProductList/ProductListpage.tsx`

- 전체 상품을 그리드(3열)로 표시
- 카테고리 및 가격 범위 필터 사이드바
- 키워드 검색 (URL 쿼리 파라미터 유지)
- 페이지네이션 (URL `?page=N` 기반)

---

## 2. 상품 상세 페이지 `/product/:id`

**파일:** `src/pages/ProductDetail/ProductPage.tsx`

- URL 파라미터로 상품 ID를 받아 데이터 조회
- 좌우 2컬럼 레이아웃 (상품 이미지/정보 + 옵션 선택)
- 사이즈 선택
- 재고 수량 표시
- 장바구니 담기

---

## 3. 로그인 페이지 `/login`

**파일:** `src/pages/LoginPage/LoginPage.tsx`

- 이메일 / 비밀번호 입력
- 비밀번호 표시/숨김 토글
- 이메일 형식 유효성 검사
- Google OAuth 로그인
- 회원가입 페이지 링크
- 로그인 상태이면 홈으로 리다이렉트 (GuestRoute)
- 실패 시 토스트 메시지 표시

---

## 4. 회원가입 페이지 `/signup`

**파일:** `src/pages/SignupPage/SignupPage.tsx`

- 이메일, 이름, 비밀번호, 비밀번호 확인 입력
- 주소 및 연락처 입력
- 이용약관 동의 체크박스
- 유효성 검사
  - 이메일 형식
  - 비밀번호 최소 길이
  - 비밀번호 일치 확인
  - 필수 항목 미입력
- 필드별 오류 메시지 표시
- 로그인 페이지 링크

---

## 5. 장바구니 페이지 `/cart`

**파일:** `src/pages/Cart/CartPage.tsx`

- 담긴 상품 목록 표시 (상품명, 이미지, 사이즈, 수량)
- 수량 변경 (재고 범위 내)
- 상품 삭제
- 빈 장바구니 안내 UI
- 주문 요약 (소계, 배송비, 총합계)
- 무료 배송 기준 금액 표시
- 재고 초과 또는 품절 상품 감지 → 결제 버튼 비활성화
- 결제 페이지로 이동 링크

---

## 6. 결제 페이지 `/payment`

**파일:** `src/pages/Payment/PaymentPage.tsx`

- 배송 정보 입력 (성, 이름, 연락처, 주소, 도시, 우편번호)
- 결제 수단 선택
  - 신용카드: 카드 번호, 소유자명, 유효기간, CVC + 카드 시각화
  - 무통장 입금: 은행 선택, 계좌번호, 입금자명
- 전체 필드 유효성 검사 (미입력 시 필드별 오류 메시지)
- 주문 요약 사이드바
- 주문 생성 후 주문 완료 페이지로 이동
- 재고 부족으로 주문 실패 시 토스트 + 장바구니로 리다이렉트

---

## 7. 주문 완료 페이지 `/payment/success`

**파일:** `src/pages/OrderComplete/OrderCompletePage.tsx`

- 결제 성공 체크 아이콘 + 감사 메시지 표시
- 주문 번호 표시
- 쇼핑 계속하기 버튼 (`/`)
- 주문 내역 보기 버튼 (`/orders`)
- 직접 URL 접근 시 홈으로 리다이렉트 (주문 번호 없으면 차단)

---

## 8. 내 주문 내역 페이지 `/orders`

**파일:** `src/pages/MyOrderList/MyOrderListPage.tsx`

- 로그인한 사용자의 주문 목록 표시
- 무한 스크롤 (Intersection Observer + 커서 기반 페이지네이션)
- 조회 기간 필터 (1개월 / 3개월)
- 스켈레톤 로딩 UI
- 빈 목록 안내 UI
- 주문별 배송 상태 배지

---

## 9. 관리자 - 재고 관리 페이지 `/admin/inventory`

**파일:** `src/pages/AdminInventoryPage/AdminInventoryPage.tsx`

- 전체 상품 목록 테이블
- 검색 및 정렬 필터 (이름/가격/등록일/수정일, 오름차순/내림차순)
- 카테고리 필터
- URL 쿼리 파라미터 기반 페이지네이션
- 상품 추가 모달 (상품명, 이미지, 카테고리, 가격, 사이즈별 재고)
- 상품 수정 모달
- 사이즈별 재고 직접 수정 (StockManager)

---

## 10. 관리자 - 주문 관리 페이지 `/admin/orders`

**파일:** `src/pages/AdminOrder/AdminOrderPage.tsx`

- 전체 주문 목록 테이블 (주문번호, 고객명, 주문 일시, 결제 금액, 연락처, 배송 상태)
- 주문번호 / 상품명 키워드 검색
- URL `?page=N` 기반 페이지네이션
- 배송 상태 변경 (결제 완료 → 배송 중 → 배송 완료 / 취소됨)
- 상세보기 버튼 → 주문 상세 모달
  - 주문번호, 주문 일시, 배송 상태
  - 고객 정보 (이름, 연락처)
  - 배송지 (주소, 도시, 우편번호)
  - 주문 상품 목록 (상품명, 사이즈, 수량, 금액)
  - 총 결제 금액

---

## 접근 제어 요약

| 유형 | 구현 | 동작 |
|------|------|------|
| 비로그인 전용 (GuestRoute) | 로그인/회원가입 | 로그인 상태이면 `/`로 리다이렉트 |
| 로그인 필요 (PrivateRoute) | 장바구니, 결제, 주문 | 미인증 시 `/login`으로 리다이렉트 |
| 관리자 전용 (PrivateRoute admin) | 재고/주문 관리 | 비관리자 접근 차단 |
| 직접 URL 접근 차단 | 주문 완료 페이지 | 주문 번호 없으면 `/`로 리다이렉트 |
